PIC32 Motor Driver

Background

This introduction to embedded systems utilizes C programming with the PIC32 to help acquaint the student with the basics of embedded programming. This includes accessesing and utilizing chip registers; creating Interrupt Service Routines to prioritize tasks; and learning how to use communication protocols like SPI, I2C, and serial.

Goal

The goal of this project was to to implement a PID system using a PIC32 microcontroller on a stepper motor with an encoder. This motor was programmed to respond to a MATLAB GUI via the command line through a serial port that controlled whether it ran, mimicked an input trajectory, or waited for user commands. Each command is clearly outlined in the video, where the GUI is displayed.

Results

The motor driver was a complete success. In the video below, I show a walkthrough how to use the GUI, and how responsive the motor is to commmands that are in input.

Conclusions

From this project, I was able to fuel my passion for mechatronics. I learned how to code in C in a way that is memory-efficient, and how to use bare metal C to communicate with chip registries in microcontrollers. I also gained a a deep understanding of communication protocols like SPI, I2C, and serial and their respective uses in industry. This understanding came from implementation of these protocols, and how they could help chips talk to one another.
